# T12 TOP3/5 游戏类型分层

## 模板用途

针对 TOP3/5 用户与非 TOP3/5 用户输出游戏类型分布。

## 建议问题（可转为 sql_pair.question）

- 针对 TOP3/5 用户与非 TOP3/5 用户输出游戏类型分布
- 对比某渠道 TOPN 与非TOPN 用户在各游戏类型上的投注分布

## 核心表/模型

- dwd_bet_order
- bds_bet_order_detail
- game_type
- game_line
- game_line_series

## 参数

- tenant_plat_id
- channel_id
- start_date
- end_date
- top_n

## SQL 模板

```sql
WITH bet_rank_base AS (
    SELECT
        b.player_id,
        SUM(b.valid_bet_amount) AS total_valid_bet_amount
    FROM dwd_bet_order b
    WHERE b.settle_status = 1
      AND b.tenant_plat_id = :tenant_plat_id
      AND b.channel_id = :channel_id
      AND b.settle_time >= :start_date
      AND b.settle_time < DATE_ADD(:end_date, INTERVAL 1 DAY)
    GROUP BY b.player_id
),
ranked_users AS (
    SELECT
        br.player_id,
        br.total_valid_bet_amount,
        ROW_NUMBER() OVER (
            ORDER BY br.total_valid_bet_amount DESC, br.player_id
        ) AS bet_rank
    FROM bet_rank_base br
),
segmented_users AS (
    SELECT
        ru.player_id,
        ru.total_valid_bet_amount,
        ru.bet_rank,
        CASE
            WHEN ru.bet_rank <= :top_n THEN CONCAT('TOP', :top_n)
            ELSE CONCAT('非TOP', :top_n)
        END AS user_segment,
        CASE
            WHEN ru.bet_rank <= :top_n THEN 1
            ELSE 2
        END AS segment_sort
    FROM ranked_users ru
),
game_type_agg AS (
    SELECT
        su.user_segment,
        su.segment_sort,
        b.game_type_id,
        COUNT(*) AS bet_times,
        COUNT(DISTINCT b.player_id) AS bet_user_count,
        SUM(b.valid_bet_amount) AS valid_bet_amount,
        SUM(b.win_loss_amount) AS win_loss_amount
    FROM bds_bet_order_detail b
    INNER JOIN segmented_users su
            ON su.player_id = b.player_id
    WHERE b.settle_status = 1
      AND b.game_type_id IS NOT NULL
      AND b.tenant_plat_id = :tenant_plat_id
      AND b.channel_id = :channel_id
      AND b.settle_time >= :start_date
      AND b.settle_time < DATE_ADD(:end_date, INTERVAL 1 DAY)
    GROUP BY su.user_segment, su.segment_sort, b.game_type_id
)
SELECT
    a.user_segment,
    a.segment_sort,
    a.game_type_id,
    gt.name AS game_type_name,
    a.bet_user_count,
    a.bet_times,
    a.valid_bet_amount,
    ROUND(a.valid_bet_amount / NULLIF(a.bet_times, 0), 4) AS avg_bet_amount,
    a.win_loss_amount,
    ROUND(a.win_loss_amount / NULLIF(a.valid_bet_amount, 0), 6) AS kill_rate,
    ROUND(
        a.valid_bet_amount / NULLIF(SUM(a.valid_bet_amount) OVER (PARTITION BY a.user_segment), 0),
        6
    ) AS bet_share_in_segment
FROM game_type_agg a
LEFT JOIN game_type gt
       ON gt.id = a.game_type_id
ORDER BY a.segment_sort, a.valid_bet_amount DESC, a.game_type_id;
```

## 备注

- 分层逻辑复用 `T06`：按整个统计区间累计有效投注排名，先打 TOPN 标签，再统计各游戏类型分布。
- 当前模板默认同时返回 `TOPN` 与 `非TOPN` 两组结果，便于直接对比。
- 若后续需要继续下钻到线路/系列，可在本模板上继续补 `game_line_id / game_line_series_id` 分组字段。
