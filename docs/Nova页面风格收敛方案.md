# Nova 页面风格收敛方案

更新时间：2026-04-09

## 1. 目标

将当前 Nova 前端页面从“视觉包装偏重的定制 SaaS 风”收敛为：

- **参考 `docs/refer_dula` 的版式和气质**
- **保留 Nova 品牌名与必要的紫色品牌识别**
- **整体更接近 Ant Design 企业产品风格**
- **降低视觉噪音、减少容器层级、强化主任务路径**

---

## 2. 目标风格关键词

### 2.1 `refer_dula` 风格关键词

- 清晰
- 克制
- 白底工作台
- 轻品牌化
- 结构优先
- 企业级产品界面

### 2.2 Nova 应承接的表达

- 专业可信
- 高效确定
- 路径明确
- 不炫技
- 少解释，多行动

---

## 3. 对 `docs/refer_dula` 的风格总结

## 3.1 总体风格

`refer_dula` 不是营销页风格，也不是强装饰的 SaaS 视觉风格，而是：

> **Ant Design 企业产品范式 + 品牌化皮肤**

其特点：

1. **大面积白底 / 浅灰底**
2. **紫色仅作为重点强调色**
3. **页面结构稳定，单页主任务非常明确**
4. **卡片只是内容组织工具，不是视觉主角**
5. **控件语义标准，像成熟 B 端产品**

## 3.2 页面共性

### 新对话页
- 视觉中心是输入区域
- 说明文案很短
- 推荐内容作为次级补充
- 首屏焦点集中

### 知识库页
- 更像“列表 / 工作台 / 详情”结构
- 不是展示型海报布局
- 左侧选择，右侧详情，层级清晰

### 对话结果页
- 内容主体直接展开
- 工具和上下文是辅助，不喧宾夺主
- 图表与解释内容属于主内容流，不做过度包装

### 看板页
- 标准企业 BI 工作台结构
- 筛选、指标、图表明确分区
- 不依赖重背景或重卡片强化层次

---

## 4. Ant Design 风格判断

## 4.1 `refer_dula` 是否符合 Ant Design 气质

**结论：符合。**

虽然不是 Ant Design 默认皮肤，但其设计语言与 AntD 企业产品气质高度一致：

- 布局理性
- 控件标准
- 主次操作分明
- 信息架构优先
- 装饰性克制

## 4.2 当前 Nova 是否符合 Ant Design 气质

**结论：部分符合，但整体偏离。**

偏离点主要不在组件来源，而在视觉策略：

- AntD 组件被重度二次包装
- 全局视觉 override 过强
- 页面自定义容器过多
- 层级表达依赖圆角 / 阴影 / 氛围背景

---

## 5. 当前实现的主要问题

## 5.1 整体背景和氛围过重

当前 `wren-ui/src/styles/themes/nova.less` 中存在：

- 多层 `radial-gradient`
- 大面积氛围背景
- 全局 blur / shadow / gradient

问题：

- 页面从“工作台”变成了“包装页”
- 背景在参与表达，抢走了结构的注意力

### 直接证据
- `nova.less` 存在多层 `radial-gradient` 与 `linear-gradient`
- `primary button` 使用渐变背景
- `modal` 使用 `backdrop-filter`

## 5.2 壳层过于自定义

当前 `wren-ui/src/components/reference/DolaAppShell.tsx` 里：

- sidebar 自带重容器
- nav / history / footer 都是独立卡块
- 存在多处渐变、阴影、大圆角、blur

问题：

- 左侧不像一个连续工作区
- 更像多个展示卡片拼起来
- 不像成熟企业产品侧栏

## 5.3 首页首屏信息过密

当前首页首屏同时出现：

- 页面标题
- 副标题
- hint 文案
- meta 卡
- alert
- scope chips
- 模式切换
- 技能入口
- composer 状态块
- 推荐问题

问题：

- 焦点太多
- 用户 3 秒内无法判断唯一主任务
- 信息密度高于参考图

## 5.4 知识库页层级过多

当前知识库页表现为：

- 顶部 hero 容器
- 左侧列表容器
- 右侧详情容器
- 下方资产空态容器

问题：

- 容器之间都在争夺视觉边界
- 不像 Dula 的“列表 + 详情工作台”

## 5.5 控件语义不够标准

现在很多区域依赖自定义：

- 自定义 chips
- 自定义 pill
- 自定义按钮
- 自定义 section 容器

问题：

- 用户不容易快速识别控件性质
- 也不符合 AntD 的“控件语义稳定”优势

---

## 6. 需要删掉的设计倾向

以下是收敛阶段必须压缩或删除的内容：

1. **整站大面积氛围背景**
2. **重阴影**
3. **过大的圆角**
4. **按钮和品牌块的渐变依赖**
5. **玻璃态 / blur 作为常规视觉语言**
6. **卡片套卡片**
7. **一个页面内多个同级强调块**
8. **说明文案和辅助提示过多**

---

## 7. 必须保留的内容

1. **Nova 品牌名**
2. **紫色作为主品牌强调色**
3. **左侧固定导航**
4. **Workspace 作为第一上下文**
5. **新对话 / 知识库 / 工作空间 / 数据看板 / 设置 的一级导航结构**
6. **问答、知识库、工作空间、看板四大任务分区**

---

## 8. 风格收敛原则

## 8.1 结构优先于装饰

先让用户知道：
- 我在哪
- 我现在能做什么
- 下一步在哪里

再做视觉优化。

## 8.2 一页只保留一个主任务

- 新对话：发起问答
- 知识库：浏览/管理知识库
- 工作空间：切换/查看工作空间
- 数据看板：浏览看板

## 8.3 轻容器、弱背景、强路径

- 背景尽量退后
- 容器边界尽量减少
- 主按钮与主内容路径清楚

## 8.4 尽量回到 AntD 原生语义

优先使用：

- `Layout`
- `Menu`
- `Button`
- `Input`
- `Tabs`
- `Segmented`
- `Dropdown`
- `Modal`
- `Drawer`
- `Table`
- `Card`（仅在真正需要时）

避免同一语义重复造一层视觉壳。

---

## 9. 页面级收敛方案

## 9.1 登录页

### 目标
- 像标准企业登录页
- 干净、直接、可立即操作

### 调整方向
- 保留左侧简短品牌介绍
- 右侧登录卡进一步简化
- 去掉多余品牌光效和复杂背景
- 使用更标准的表单控件边框、间距、按钮样式

### 要点
- 背景改为纯浅灰或极轻渐变
- 登录卡阴影减弱
- 输入框改回更接近 AntD 原生样式
- 主按钮保留品牌紫，但去掉“悬浮营销感”

## 9.2 左侧导航系统

### 目标
从“分段卡片栏”改成“连续工作区边栏”。

### 现状问题
- 导航卡、历史卡、底部卡是分裂的
- 底部设置/退出太像附加信息，不像系统入口

### 调整方向
- Sidebar 只保留一个连续背景层
- 导航区、历史区、底部区通过间距和分割线区分
- 不再给每个区块独立强卡片感

### 具体要求
- 去掉导航卡和历史卡的重阴影
- 降低圆角
- 历史搜索和历史列表直接嵌入边栏流中
- 设置 / 退出作为底部系统操作，弱化装饰、强化语义

## 9.3 新对话页

### 目标
把首屏收成“一个主任务”。

### 现状问题
- 说明太多
- 首屏块数太多
- composer 上下文过于复杂

### 调整方向
- 保留标题，但缩短副标题与 hint
- 删除 meta 卡与 alert 的组合使用
- 上下文选择区整合进一个主输入容器
- 推荐问题放在输入区下方，弱化其视觉重量

### 推荐结构
```text
标题
一句话说明

[ 知识库选择 / 模式切换 / 技能入口 / 输入框 / 发送 ]

推荐问题
```

### 替换建议
- 将自定义 chip/切换器尽量替换成 AntD `Segmented` / `Select` / `Tag`
- 将输入区做成单一主卡，而不是“外卡 + 内卡 + 提示条”

## 9.4 知识库页

### 目标
回到标准“左侧列表 + 右侧详情”工作台。

### 现状问题
- 顶部 hero 太大
- 中区容器太多
- 空态卡过强

### 调整方向
- 顶部只保留：标题 + 一句话说明 + 主按钮
- 左侧是知识库列表与筛选
- 右侧是当前知识库详情
- 下方内容按 Tab 或分段自然展开

### 替换建议
- 大量自定义区块收敛为：
  - `Tabs`
  - `List`
  - `Descriptions`
  - `Empty`
  - `Table`
- “添加资产 / 分析规则 / SQL 模板”放入右上操作区，不要四处散落

## 9.5 工作空间页

### 目标
像租户/空间管理页，而不是介绍页。

### 调整方向
- 顶部：标题 + 当前 workspace 状态 + 主操作
- 内容区：我加入的工作空间列表 / 成员 / 说明
- 避免大段介绍文案
- 切换器与管理动作分层

## 9.6 数据看板页

### 目标
更像标准 BI 页面。

### 调整方向
- 顶部：标题 + 时间范围 / 筛选器 / 操作按钮
- 中间：指标行
- 下方：图表网格

### 禁止
- 图表外层过度包装
- 每个图表都加重卡片阴影
- 页面背景参与图表层级表达

## 9.7 弹窗 / 抽屉 / 表单

### 目标
回到标准企业产品交互。

### 调整方向
- Modal 白底、轻阴影、轻圆角
- 表单使用标准 label + input + action
- 禁止在 modal 中叠加太多说明卡
- 复杂流程改抽屉或分步，不靠大段文字解释

## 10. 视觉 Token 收敛建议

## 10.1 背景
- 页面背景：从当前多层氛围渐变，改为**纯浅灰 / 极轻线性渐变**
- 主内容背景：白色
- 辅助容器背景：`#fafafa` / `#fcfcfd`

## 10.2 圆角
- 当前偏大
- 目标：
  - 页面主容器：16~20
  - 常规卡片：12~16
  - 输入框/按钮：8~12

## 10.3 阴影
- 从“可见阴影”改为“弱感知阴影”
- 主要依赖边框和留白建立层次

## 10.4 品牌色
- 紫色保留
- 只用于：
  - 主按钮
  - 当前选中
  - 标签点缀
  - 焦点态

不要让紫色参与大面积背景渲染。

---

## 11. 组件策略

## 11.1 保留可复用的壳层结构

可以保留：
- `DolaAppShell` 的信息架构
- `RuntimeScopeSelector` 的业务能力
- 现有导航路径和路由结构

## 11.2 需要重写/收敛的部分

优先重写或简化：

1. `wren-ui/src/styles/themes/nova.less`
2. `wren-ui/src/components/reference/DolaAppShell.tsx`
3. `wren-ui/src/pages/home/index.tsx`
4. `wren-ui/src/pages/knowledge/index.tsx`

## 11.3 优先回归 AntD 原生表达的区域

- 按钮
- 输入框
- Tab
- 空态
- Modal
- 列表
- 筛选器
- 顶部操作区

---

## 12. 实施波次

## Wave 1：视觉基础收敛

目标：先把“复杂感”压下去。

任务：
1. 收敛 `nova.less` 全局主题
2. 去掉重背景、重阴影、blur
3. 降低圆角
4. 统一按钮 / 输入框 / modal 的 AntD 化表达

## Wave 2：导航壳层收敛

目标：让左侧像连续工作区。

任务：
1. 重构 `DolaAppShell`
2. 去除导航卡/历史卡/底部卡的割裂感
3. 固定左栏，统一边栏内部层级

## Wave 3：首页收敛

目标：让新对话页回到一个主任务。

任务：
1. 删除冗余说明块
2. 合并上下文选择和输入区
3. 弱化推荐问题视觉比重

## Wave 4：知识库页收敛

目标：回到“列表 + 详情”工作台。

任务：
1. 压缩顶部 hero
2. 收敛中部多容器结构
3. 强化右侧详情主区

## Wave 5：工作空间 / 数据看板统一

目标：建立统一企业产品页面标准。

任务：
1. 工作空间页标准化
2. 数据看板页标准化
3. 所有弹窗/抽屉/空态统一

---

## 13. 验收标准

收敛完成后，应满足：

1. 用户打开页面后，**3 秒内知道主任务是什么**
2. 首屏只存在 **一个主视觉焦点**
3. 页面背景不再参与主要层级表达
4. 左侧导航看起来像一个连续工作区，而不是多个拼接卡片
5. 页面中可见容器数量明显下降
6. 大多数常见交互控件看起来像标准企业产品控件
7. 整体气质接近 `docs/refer_dula`，但仍保留 Nova 品牌紫色识别

---

## 14. 当前建议的执行顺序

建议直接按以下顺序推进：

1. **先改 `nova.less`**
2. **再改 `DolaAppShell`**
3. **然后收首页**
4. **再收知识库页**
5. **最后统一工作空间 / 看板 / 弹窗**

原因：
- 如果不先收主题和壳层，后续页面级修改会反复返工。
