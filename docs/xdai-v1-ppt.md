## Slide 2 — 产品定位

AI 数据分析平台是一个面向企业的 **生成式 BI 平台**。

用户用自然语言提问，平台自动查询数据、生成 SQL、返回结果与图表。

**核心场景**

- **业务人员**：用自然语言提问，平台自动理解意图、查询数据、生成图表，无需写 SQL
- **技术团队**：接入数据源构建知识库，将自定义数据能力封装成场景 Skill，供业务人员开箱即用
- **运营团队**：将查询结果固化到看板持续观测，或配置定时报表自动推送

## Slide 3 — 产品结构

```
Workspace（租户边界）
  ├── 成员管理 & 角色权限
  ├── 知识库（Knowledge Base）
  │    ├── 数据连接（Connector）
  │    ├── 场景 Skill
  │    ├── 资产目录（表 / 视图 / API / 指标）
  │    └── 知识资产
  │         ├── 术语表（Glossary）
  │         ├── 分析规则（Analysis Rules）
  │         ├── SQL 模板
  │         ├── SQL 示例对
  │         └── 使用说明（Instruction）
  ├── 数据看板（多个，绑定具体知识库执行）
  └── 定时任务（绑定具体知识库执行）
```

## Slide 4 — 知识库

**知识库**是用户面对的核心运营对象。

每个 Workspace 下可管理多个知识库，每个知识库独立维护自己的：
- 数据连接与资产（表、视图、API、指标）
- 知识资产（术语表、分析规则、SQL 模板、SQL 示例对、使用说明）
- 场景 Skill

**知识库隔离**
- 不同知识库的数据、规则、对话上下文互不干扰
- 对话（Thread）创建时绑定到具体知识库，全程不切换

## Slide 5 — 问数编排流程

```
用户提问
    │
    ▼
装配知识库上下文（术语 / 规则 / 模板）
    │
    ▼
SkillRouter
    ├── 命中场景 Skill → Isolated Runner 执行
    │       └── 返回结构化结果
    └── 未命中 → SQL 语义引擎 NL2SQL
            └── SQL 生成 → 执行 → 返回结果
    │
    ▼
MixedAnswerComposer → 最终回答 + 图表
```

- 支持 Skill 结果与 SQL 结果混合组合
- 任何路径失败可以回退
- 全链路 trace，每步可观测

## Slide 6 — 场景 Skill

**Skill 适用于相对确定的业务场景**（固定报表、ROI 计算、标准看板），绕过 LLM 直接执行已知查询逻辑，保证结果的确定性与稳定性。

NL2SQL 路径处理探索性问题，知识资产（术语表、分析规则、SQL 模板）辅助提升生成精度。高频验证稳定的查询可进一步沉淀为 Skill，形成**探索 → 固化 → 提速**的持续优化飞轮。

场景 Skill 是用户编写的执行单元，**系统不限制它的实现方式**：

| Skill 类型 | 说明 |
|---|---|
| API Skill | 调用外部/内部 REST API |
| DB Skill | 直接查询数据库 |
| 复合 Skill | 多步 Agent 流程，先 API 再聚合 |

**系统保证**
- 隔离执行（独立 worker，不影响主服务）
- 注入用户身份、加密凭证、连接配置
- 统一输出格式：表格 / 指标 / 文本 / 图表
- 完整审计日志 + 超时与资源限制

---

## Slide 7 — 语义引擎 NL2SQL

平台内建 **SQL 语义引擎**，将自然语言问题转化为可执行 SQL。

**工作方式**
1. 用户提问经过意图分类，判断是否走 NL2SQL 路径
2. 从知识库中检索相关 schema、SQL 示例、使用说明作为上下文
3. LLM 结合上下文生成 SQL
4. SQL 在语义层校验并执行，失败时自动纠错重试
5. 返回结构化结果，可进一步生成图表

**精度增强机制**
- 术语表：确保业务词汇映射到正确字段
- 分析规则：注入业务计算逻辑
- SQL 模板：复用已验证的高频查询模式
- SQL 示例对：通过历史样例提升生成准确率

---

## Slide 8 — 账号与权限

**账号体系**
- 支持账号（邮箱 + 密码），SSO / OIDC
- 首个用户自动成为 Owner，支持邀请成员

**权限层级**
- Workspace 级：成员可见哪些知识库
- 知识库级：成员能否读、写、管理
- 支持行列级权限

**Deny-by-default**：缺少授权时默认拒绝，不静默放行

```
│  PostgreSQL + pgvector（向量检索）   │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│  SQL 语义引擎                        │  语义模型 + 多数据源执行
└─────────────────────────────────────┘
```
