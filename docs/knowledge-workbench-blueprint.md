# 知识库工作台改版蓝图

更新时间：2026-04-16

## 1. 目标

把当前 `/knowledge` 从“知识库详情页 + 多个弹窗入口”升级为真正的**知识库工作台**：

- 知识库 = 一个完整工作对象
- SQL 模板 / 分析规则 / 建模 / 资产 / 发布快照都尽量在同一工作台中完成
- 弹窗仅保留给轻量输入、确认操作和详情编辑

## 2. 信息架构

### 左侧

- 知识库列表
- 知识库筛选
- 创建知识库

### 主区

以工作台分区承载核心能力：

- 概览
- 资产
- 建模
- SQL 模板
- 分析规则
- 发布 / 快照

### 弹窗保留范围

- 新建 / 编辑知识库
- 轻量详情编辑
- 危险操作确认

## 3. 交互原则

1. **主流程不再依赖管理弹窗**
   - SQL 模板列表管理
   - 分析规则列表管理
   - 建模入口
   应该作为工作台主区内容出现

2. **上下文始终锁定到当前知识库**
   - 用户在工作台内明确知道“我在建设哪个知识库”

3. **重操作优先主区 / 子工作区**
   - 例如建模应进入知识库内的建模分区或对应的完整工作区

## 4. 分阶段落地

### Phase A（当前实施）

- `/knowledge` 主区引入工作台分区
- SQL 模板 / 分析规则从“管理弹窗主入口”迁回主区
- 建模作为工作台主区分区出现，并提供进入完整建模工作区的入口
- 发布 / 快照先提供状态面板

### Phase B（后续）

- 建模区进一步内嵌更多直接操作能力
- 发布 / 快照补全发布检查、差异、执行结果
- 资产配置从向导式弹窗进一步迁移到工作台侧栏 / 主区

## 5. 当前实现约束

- 现有 SQL 模板 / 分析规则详情编辑仍复用 detail modal
- 现有完整建模能力仍复用 `/modeling` 工作区
- 现阶段优先解决“主入口与上下文”问题，而不是一次重写所有深层编辑器

## 6. 完成标准

以下条件满足，认为知识库工作台第一阶段完成：

1. 用户进入 `/knowledge` 后能在主区直接看到工作台分区
2. SQL 模板 / 分析规则列表不再必须通过管理弹窗进入
3. 建模成为知识库工作台的一部分，而不是游离在知识库上下文之外
4. 弹窗仅承担轻量编辑与确认，不再承担主导航角色
