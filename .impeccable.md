## Design Context

### Users
- 主要用户：企业内部的数据分析师、业务运营、管理者（非技术用户占多数）。
- 使用场景：在日常业务会议前后、复盘与临时追问时，通过自然语言快速拿到可解释的数据结论与图表。
- 核心任务：
  1. 快速发起问答并拿到可信答案；
  2. 在知识库中维护数据资产、规则、SQL 模板；
  3. 在 workspace 隔离下安全协作与复用。

### Brand Personality
- 品牌关键词：专业可信、清晰克制、高效确定。
- 语气与感受：
  - 让用户“知道下一步该做什么”；
  - 不炫技，不堆信息；
  - 关键动作明确、反馈及时、错误可恢复。

### Aesthetic Direction
- 产品名：Nova。
- 视觉方向：参考 `docs/refer_dula` 的配色与版式风格，但统一到 Nova 品牌表达。
- 信息架构：严格遵循 `docs/前端交互蓝图与任务拆解.md`。
- 导航结构：
  1. 新对话
  2. 我的知识库
  3. 工作空间
  4. 数据看板
  5. 设置（左下）
- 设计限制：避免“卡片套卡片”、伪按钮、无反馈点击；所有交互必须有状态回执（跳转/弹窗/加载/禁用原因）。

### Design Principles
1. **路径优先**：每页只保留一个主任务，用户 3 秒内能判断“这页能做什么”。
2. **上下文先行**：Workspace 是第一上下文；知识库、对话、看板都在当前 workspace 内完成。
3. **范围锁定**：Thread 在创建时固化 `workspace + knowledgeBaseIds[]`，不中途切换知识库范围。
4. **最少认知负担**：移除无用说明与重复入口，减少视觉噪音，保留高价值决策信息。
5. **状态完整**：主流程必须覆盖正常/空态/loading/error/禁用说明，避免“点了没反应”。
6. **一致但不僵化**：统一间距、层级、控件语义；在关键区域（首页首屏、关键 CTA）建立可识别视觉记忆点。
